EXPONENTIAL SEARCH
Step 1: Given a sorted array arr[] and a target element x.

Step 2: If the first element of arr[] is equal to x, return 0 (target found).

Step 3: Set a variable 'bound' to 1 and initialize 'left' and 'right' to 0 and min(2^bound, n-1) respectively.

   (where n is the length of the array)

Step 4: While arr[right] < x and right < n:

    a. Update 'left' to 'right'.

    b. Double the value of 'bound'.

    c. Update 'right' to min(2^bound, n-1).

Step 5: Perform a binary search between 'left' and 'right':

    a. If 'left' is greater than 'right', the target element is not present in the array. Return -1 (target not found).

    b. Set a variable 'mid' to the middle index between 'left' and 'right'.

    c. If arr[mid] is equal to x, return 'mid' (target found).

    d. If arr[mid] is less than x, update 'left' to 'mid + 1' and repeat step 5.

    e. If arr[mid] is greater than x, update 'right' to 'mid - 1' and repeat step 5.

Step 6: If the algorithm hasn't returned by this point, the target element is not present in the array. Return -1 (target not found).

TERNARY SEARCHING
Step 1: Given a sorted array arr[] and a target element x.

Step 2: Set variables 'left' to 0 and 'right' to n-1, where n is the length of arr[].

Step 3: While 'left' is less than or equal to 'right':

	     a. Set 'mid1' to 'left' + floor((right - left) / 3).

	     b. Set 'mid2' to 'right' - floor((right - left) / 3).

	     c. If arr[mid1] is equal to x, return 'mid1' (target found).

	     d. If arr[mid2] is equal to x, return 'mid2' (target found).

	     e. If x is less than arr[mid1], update 'right' to 'mid1 - 1'.

	     f. If x is greater than arr[mid2], update 'left' to 'mid2 + 1'.

	     g. Otherwise, set 'left' to 'mid1 + 1' and 'right' to 'mid2 - 1'.

Step 4: If the algorithm hasn't returned by this point, the target element is not present in the array. Return -1 (target not found).
